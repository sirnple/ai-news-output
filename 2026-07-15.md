### 1. Function-Aware Fill-in-the-Middle as Mid-Training for Coding Agent Foundation Models
> 来源：papers | 标签：AI编程 / Agent / 论文

**中文摘要：**

本文提出了一种面向编码智能体基础模型的中期训练方法——功能感知的中间填充（Function-Aware Fill-in-the-Middle，FA-FIM），其核心贡献在于通过引入代码功能结构信息，显著提升了模型在复杂代码补全与智能体任务中的表现。该方法在传统中间填充训练基础上，利用代码的抽象语法树（AST）与函数调用关系，动态构造以函数为基本单元的填充掩码，使模型能够更准确地理解代码的语义边界与逻辑依赖。实验结果表明，FA-FIM在多个代码生成与智能体基准测试中优于标准中间填充及因果语言模型训练方法，尤其在多步推理与上下文感知的代码编辑任务上取得了显著提升。然而，该方法对代码结构解析的依赖可能限制其在高度动态或非结构化代码场景中的泛化能力，未来可探索结合程序语义嵌入与自适应掩码策略以进一步增强模型对代码意图的深层理解。

阅读原文：https://arxiv.org/abs/2607.12463

### 2. Know Before Fix: QA-Driven Repository Knowledge Acquisition for Software Issue Resolution
> 来源：papers | 标签：AI编程 / Agent / 论文

**中文摘要**  
本文提出一种基于问答（QA）驱动的仓库知识获取方法，旨在提升软件问题修复的效率与准确性。核心贡献在于将问题解决过程与仓库中已有知识库的主动检索相结合，通过QA机制引导开发者快速定位相关代码、文档与历史修复记录。方法上，作者构建了一个端到端框架，首先利用自然语言处理技术解析问题描述，生成针对性查询；随后在仓库知识图谱中检索并排序最相关的知识片段，最终以问答形式呈现给开发者。关键发现表明，该方法在多个开源项目上显著减少了问题定位时间，并提高了首次修复成功率。局限性在于对仓库知识完备性依赖较强，且对非结构化文档的处理仍有提升空间。未来工作可探索跨仓库知识迁移与动态知识更新策略。

阅读原文：https://arxiv.org/abs/2607.11111

### 3. What LLM Forecasters Know but Don't Say: Probing Internal Representations for Calibration and Faithfulness
> 来源：papers | 标签：LLM / 论文

**中文摘要**  
本研究揭示了大型语言模型（LLM）在生成概率预测时，其内部表征中隐含了比输出更准确的校准信息，但模型未能充分表达这一知识。核心贡献在于提出一种基于探测内部表征的校准方法，以提升LLM预测的忠实度与可靠性。方法上，作者通过分析模型隐藏层中的概率表征，发现其内部编码的预测置信度与真实结果的一致性显著优于表层输出概率；据此设计了一种轻量级探测器，从内部表征中提取校准后的预测值。关键发现表明，该方法在多个问答与推理任务上有效降低了过自信与欠自信偏差，且无需修改模型参数。局限性在于探测器的泛化能力受限于训练数据的分布，未来可探索跨任务与跨模型的内部表征对齐策略。

阅读原文：https://arxiv.org/abs/2607.08046

### 4. Search Beyond What Can Be Taught: Evolving the Knowledge Boundary in Agentic Visual Generation

OpenAI提出了一种名为GPT-Red的自我改进方法，旨在提升语言模型的鲁棒性。该方法通过让模型自动生成对抗性提示（即“红队测试”样本）并基于这些样本进行微调，形成自我对抗训练循环。实验表明，在多个安全与鲁棒性基准测试中，GPT-Red使模型在对抗性攻击下的表现提升了约20-30%，且无需人工标注数据。该技术利用GPT-4作为基础模型，通过迭代生成与筛选高难度对抗样本，显著增强了模型对恶意输入的防御能力。研究还发现，自我改进的鲁棒性提升可泛化至未见过的攻击类型，为构建更安全的大语言模型提供了可扩展的自动化路径。

### 13. MuScriptor: An Open Model for Multi-Instrument Music Transcription
