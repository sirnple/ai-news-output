### 1. Show HN: Reverse-engineering web apps into agent tools
> 来源：hackernews | 标签：Agent

该文章介绍了一种将Web应用逆向工程转化为AI代理工具的方法。核心思路是通过分析Web应用的网络请求、DOM结构和API接口，自动生成可供AI代理调用的结构化工具描述（如OpenAPI规范）。关键细节包括：使用浏览器开发者工具或代理抓包获取请求参数与响应格式，结合DOM解析识别交互元素（如按钮、表单），最终生成包含输入输出模式、认证方式等元数据的工具定义。该方法可显著降低手动编写工具适配代码的工作量，支持将现有SaaS应用（如Slack、Notion）快速接入AI工作流。文中提及一个开源原型工具，能在5分钟内完成对典型CRUD应用的逆向建模，并支持OAuth2.0认证的自动处理。

阅读原文：https://news.ycombinator.com/item?id=48847834

### 2. Show HN: Frugon – Find which LLM calls a cheaper model could handle (local, MIT)
> 来源：hackernews | 标签：LLM

Frugon 是一个本地运行的 MIT 开源工具，旨在通过分析 LLM 调用历史，识别出哪些请求可以由更便宜的模型处理，从而降低 API 成本。它基于用户提供的 OpenAI 格式日志，利用嵌入相似度与成本阈值，自动筛选出可替换为廉价模型的调用。该工具完全本地运行，不发送数据至外部，支持自定义模型组合与成本参数。核心功能包括：分析历史调用、生成替换建议、输出可执行的配置。适用于需要优化 LLM 调用成本但不愿牺牲关键任务质量的开发团队。

阅读原文：https://github.com/Rodiun/frugon

### 3. Remember When It Matters: Proactive Memory Agent for Long-Horizon Agents
> 来源：papers | 标签：Agent

**中文摘要：**  
本文提出一种名为“主动记忆智能体”（Proactive Memory Agent）的新型框架，旨在解决长时域任务中智能体记忆利用的滞后性问题。其核心贡献在于设计了一种能够根据任务上下文主动触发记忆检索与更新的机制，而非被动等待查询。方法上，该智能体通过引入“记忆重要性评估模块”与“前瞻性检索策略”，在任务执行过程中动态识别关键信息并提前激活相关记忆，从而显著提升长序列决策的连贯性与效率。实验结果表明，在多个长时域基准任务中，该方法在任务完成率与记忆利用率上均优于现有被动记忆模型。局限性在于当前框架对记忆存储规模增长时的检索开销控制尚不完善，未来可探索分层记忆结构与自适应压缩策略以进一步提升可扩展性。

阅读原文：https://arxiv.org/abs/2607.08716

### 4. venkata-murari-sunkara/GroundTruth-AI: Production-style LLM benchmarking platform for evaluating GPT-4o Mini, Gemini 2.5 Flash, and Claude Haiku 4.5 on custom benchmark datasets.
> 来源：github_trending | 标签：LLM / 论文

该文章介绍了名为 **GroundTruth-AI** 的生产级大语言模型（LLM）基准测试平台，由开发者 venkata-murari-sunkara 在 GitHub 上开源。平台专注于对 **GPT-4o Mini、Gemini 2.5 Flash 和 Claude Haiku 4.5** 等轻量级模型进行自定义数据集评估。其核心功能包括：支持用户上传私有测试集、自动化运行模型推理并生成对比报告，涵盖准确率、延迟、成本等关键指标。平台采用模块化架构，便于扩展新模型与评测维度，旨在为开发者提供贴近实际业务场景的模型选型依据，而非依赖通用排行榜。项目代码与文档均公开，便于社区复现与定制。

阅读原文：https://github.com/venkata-murari-sunkara/GroundTruth-AI

### 5. satya66123/Multi-AI-Agent: A modular AI Agent application built with Python, Streamlit, and Ollama, featuring Chat, Text Summarization, Calculator, and Weather agents in a single interactive interface.
> 来源：github_trending | 标签：Agent

该文章介绍了一个名为 **Multi-AI-Agent** 的开源项目，由开发者 satya66123 创建。项目基于 **Python**、**Streamlit** 和 **Ollama** 构建，提供了一个模块化的多智能体交互界面。核心功能包括：**聊天（Chat）**、**文本摘要（Text Summarization）**、**计算器（Calculator）** 和 **天气查询（Weather）** 四个智能体。所有智能体集成在单一界面中，用户可通过 Streamlit 前端与不同 AI 功能交互，Ollama 负责本地模型推理。项目代码开源在 GitHub，适合开发者快速搭建多智能体应用原型。

阅读原文：https://github.com/satya66123/Multi-AI-Agent
